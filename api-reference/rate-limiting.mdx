Para garantir que a API da Zynx Wallet permaneça rápida e disponível para todos, aplicamos limites de taxa (Rate Limiting) baseados na sua chave de API.

### Limites Padrão

Os limites são aplicados por minuto e variam de acordo com o tipo de operação:

| Operação | Limite |
| :--- | :--- |
| **Leitura (GET)** | 100 requisições / min |
| **Escrita (POST)** | 50 requisições / min |

### Cabeçalhos de Controle

Você pode monitorar seu uso atual através dos cabeçalhos HTTP retornados em cada resposta:

| Header | Descrição |
| :--- | :--- |
| `X-RateLimit-Limit` | O limite total permitido no período. |
| `X-RateLimit-Remaining` | Quantas requisições você ainda pode fazer. |
| `X-RateLimit-Reset` | Tempo em segundos até o limite ser resetado. |

### Quando o limite é excedido

Se você ultrapassar o limite, a API retornará um erro `429 Too Many Requests`.

```json
{
  "success": false,
  "error": "Rate limit exceeded. Please try again later."
}
```
