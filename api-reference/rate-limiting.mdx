---
title: "Rate Limiting"
description: "Limites de requisições para garantir a estabilidade da API."
---

Para garantir a performance e disponibilidade para todos os usuários, a API da Zynx Wallet aplica limites de taxa (rate limiting).

### Limites Padrão

| Categoria | Limite |
| :--- | :--- |
| **Leitura (GET)** | 100 requisições por minuto |
| **Escrita (POST)** | 50 requisições por minuto |

### Cabeçalhos de Resposta

Quando você atinge o limite, a API retornará o status `429 Too Many Requests` e incluirá os seguintes cabeçalhos:

- `X-RateLimit-Limit`: O número máximo de requisições permitidas.
- `X-RateLimit-Remaining`: O número de requisições restantes no período atual.
- `X-RateLimit-Reset`: O tempo em segundos até que o limite seja resetado.

<Tip>
  Se você precisar de limites maiores para o seu negócio, entre em contato com nosso suporte comercial.
</Tip>
